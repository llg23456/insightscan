# InsightScan 环境准备指南

> 本指南面向 **Ubuntu 22.04 虚拟机** 用户，帮助你在运行项目前完成所有环境配置。
>
> 阅读时间：约 10 分钟

---

## 一、检查清单（运行前必读）

在运行 InsightScan 之前，请确认以下 6 项已就绪：

| 序号 | 检查项 | 验证命令 | 预期结果 |
|------|--------|---------|---------|
| 1 | Ubuntu 系统版本 | `lsb_release -a` | `Ubuntu 22.04.x LTS` |
| 2 | Python 版本 | `python3 --version` | `Python 3.10.x` |
| 3 | Nmap 已安装 | `nmap --version` | 显示 `Nmap version 7.80` 或更高 |
| 4 | pip 可用 | `pip3 --version` | 显示 pip 版本号 |
| 5 | 网络连通 | `ping baidu.com` | 能收到回复 |
| 6 | 共享文件夹挂载（VMware 用户） | `ls /mnt/hgfs/` | 显示 `insightscan` 或你的项目名 |

如果以上任何一项不通过，请按下方步骤修复。

## 二、系统环境安装（首次配置）

### 2.1 更新系统

```bash
sudo apt update && sudo apt upgrade -y
```

### 2.2 安装 Python 和基础工具

```bash
sudo apt install python3 python3-pip python3-venv git -y
```

验证：
```bash
python3 --version    # 应显示 Python 3.10.12
pip3 --version       # 应显示 pip 版本
```

### 2.3 安装 Nmap（核心依赖）

```bash
sudo apt install nmap -y
```

验证：
```bash
nmap --version       # 应显示 Nmap version 7.80 或更高
```

### 2.4 安装 VMware 工具（共享文件夹用）

如果你使用 VMware 虚拟机，必须安装此工具才能访问 Windows 共享文件夹：

```bash
sudo apt install open-vm-tools open-vm-tools-desktop -y
```

安装后**重启虚拟机**：
```bash
sudo reboot
```

## 三、项目获取方式

### 方式一：Git 克隆（推荐）

```bash
# 进入你想放项目的位置（建议家目录）
cd ~

# 克隆仓库
git clone https://github.com/llg23456/insightscan.git

# 进入项目目录
cd insightscan
```

### 方式二：共享文件夹（开发模式）

如果你在 Windows 上用 Cursor/TRAE 写代码，通过 VMware 共享文件夹同步到 Ubuntu：

```bash
# 检查共享文件夹是否挂载
ls /mnt/hgfs/

# 应该看到 insightscan 文件夹
# 如果没有，手动挂载：
sudo mkdir -p /mnt/hgfs/insightscan
sudo vmhgfs-fuse .host:/insightscan /mnt/hgfs/insightscan -o allow_other -o uid=1000

# 进入项目
cd /mnt/hgfs/insightscan
```

> **注意**：共享文件夹不支持创建 Python 虚拟环境（venv），依赖请直接安装到系统环境。

## 四、Python 依赖安装

### 4.1 安装项目依赖

```bash
# 进入项目根目录（根据你的获取方式选择）
cd ~/insightscan          # Git 克隆方式
cd /mnt/hgfs/insightscan  # 共享文件夹方式

# 安装依赖
pip3 install -r requirements.txt
```

如果 `requirements.txt` 不存在或安装失败，手动安装核心依赖：

```bash
pip3 install python-nmap openai requests markdown flask matplotlib psutil
```

### 4.2 验证依赖安装

```bash
python3 -c "import nmap; print('python-nmap OK')"
python3 -c "import openai; print('openai OK')"
python3 -c "import flask; print('flask OK')"
python3 -c "import matplotlib; print('matplotlib OK')"
python3 -c "import psutil; print('psutil OK')"
```

全部显示 `OK` 说明依赖安装成功。

## 五、API Key 配置（必须）

AI 分析功能需要 Kimi API Key，没有 Key 时只能使用本地规则库降级分析。

### 5.1 申请 Key

1. 访问 [Moonshot 开放平台](https://platform.moonshot.cn/)
2. 注册账号
3. 进入「API Key 管理」→「创建 API Key」
4. 复制 Key（格式：`sk-xxxxxxxxxxxxxxxx`）

### 5.2 配置到项目

```bash
# 进入项目配置目录
cd ~/insightscan/config          # 或 /mnt/hgfs/insightscan/config

# 复制模板文件
cp api_keys.json.example api_keys.json

# 编辑配置文件
nano api_keys.json
```

填入你的 Key：
```json
{
    "kimi_api_key": "sk-你的实际APIKey",
    "base_url": "https://api.moonshot.cn/v1"
}
```

保存（Ctrl+O → Enter → Ctrl+X）。

> **重要**：`api_keys.json` 已加入 `.gitignore`，不会被提交到 GitHub，请勿手动添加。

## 七、快速验证（Hello InsightScan）

完成以上所有步骤后，运行一次完整流程验证：

```bash
# 1. 扫描本机（无需 root）
python3 main.py -t 127.0.0.1 --scan-type connect --ports 22,80

# 2. 扫描 + AI 分析（需要 API Key）
python3 main.py -t 127.0.0.1 --ports 22,80 --ai-analyze

# 3. 生成报告
python3 main.py --report-format markdown --report-task 1

# 4. 查看报告
ls reports/
```

如果看到 `reports/` 目录下有 `.md` 或 `.html` 文件生成，说明环境配置成功。

## 八、常见问题

### Q1: `pip3 install` 提示权限不足

```bash
# 方案 A：加 --user 参数
pip3 install --user -r requirements.txt

# 方案 B：使用 sudo（不推荐，可能污染系统环境）
sudo pip3 install -r requirements.txt
```

### Q2: `nmap: command not found`

```bash
sudo apt install nmap -y
# 如果还是不行，检查 PATH
which nmap
```

### Q3: 共享文件夹里 `python3 -m venv venv` 报错

```bash
# 错误：Operation not supported
# 原因：VMware hgfs 不支持符号链接

# 解决：不在共享文件夹里创建 venv，直接全局安装
pip3 install python-nmap openai requests markdown flask matplotlib psutil
```

### Q4: AI 分析报错 `AuthenticationError`

```bash
# 检查 api_keys.json 是否存在且格式正确
cat config/api_keys.json

# 检查 Key 是否有效（直接 curl 测试）
curl https://api.moonshot.cn/v1/models   -H "Authorization: Bearer sk-你的Key"
```

### Q5: SYN/OS 报错 `requires root privileges`

SYN/OS 需要 root。Web 用普通用户启动，需给 **当前用户** 配置 nmap 的 sudo 免密（每人用户名不同）。

**查看用户名：**

```bash
whoami          # 例如 liyuxi、ubuntu、student01
```

**配置（推荐，自动填入当前用户）：**

```bash
echo "$(whoami) ALL=(ALL) NOPASSWD: $(which nmap)" | sudo tee /etc/sudoers.d/insightscan-nmap
sudo chmod 440 /etc/sudoers.d/insightscan-nmap
sudo -n nmap --version
python3 run_web.py
```

手动写时，把 `你的用户名` 换成 `whoami` 的结果：

`你的用户名 ALL=(ALL) NOPASSWD: /usr/bin/nmap`

勿用 `sudo python3 run_web.py`（会找不到 pip 包）。

### Q6: 如何查看本机 IP 和网段

```bash
ip addr | grep "inet "
```

示例输出：
```
inet 127.0.0.1/8 scope host lo
inet 192.168.61.128/24 brd 192.168.61.255 scope global dynamic noprefixroute ens33
```

| 字段 | 含义 | 扫描写法 |
|------|------|---------|
| `127.0.0.1/8` | 本机回环 | `127.0.0.1` |
| `192.168.61.128/24` | 局域网 IP | 单机 `192.168.61.128`，C段 `192.168.61.0/24` |

## 九、环境验证脚本（一键检查）

保存以下内容为 `check_env.sh`，运行 `./check_env.sh` 自动检查：

```bash
#!/bin/bash
echo "=== InsightScan 环境检查 ==="
echo

echo "[1/6] 系统版本:"
lsb_release -d | cut -f2

echo "[2/6] Python 版本:"
python3 --version

echo "[3/6] Nmap 版本:"
nmap --version | head -1

echo "[4/6] pip 版本:"
pip3 --version

echo "[5/6] 网络连通:"
ping -c 1 baidu.com > /dev/null 2>&1 && echo "OK" || echo "FAIL - 请检查网络"

echo "[6/6] 共享文件夹:"
ls /mnt/hgfs/ 2>/dev/null | grep insightscan && echo "OK" || echo "未挂载（如用 Git 克隆可忽略）"

echo
echo "=== Python 依赖检查 ==="
python3 -c "import nmap; print('  python-nmap OK')" 2>/dev/null || echo "  python-nmap MISSING"
python3 -c "import openai; print('  openai OK')" 2>/dev/null || echo "  openai MISSING"
python3 -c "import flask; print('  flask OK')" 2>/dev/null || echo "  flask MISSING"
python3 -c "import matplotlib; print('  matplotlib OK')" 2>/dev/null || echo "  matplotlib MISSING"
python3 -c "import psutil; print('  psutil OK')" 2>/dev/null || echo "  psutil MISSING"

echo
echo "=== API Key 检查 ==="
[ -f config/api_keys.json ] && echo "  api_keys.json 存在" || echo "  api_keys.json 不存在 - 请复制模板并配置"

echo
echo "=== 检查完成 ==="
```

运行：
```bash
chmod +x check_env.sh
./check_env.sh
```
